GitHub Pages로 만든 블로그는 빠르고 비용이 적게 듭니다. 하지만 블로그를 만들었다고 해서 검색엔진에 자동으로 잘 노출되는 것은 아닙니다. 기본적인 SEO 설정을 챙겨야 검색엔진이 사이트 구조와 글 내용을 이해할 수 있습니다.

이 글에서는 정적 블로그 운영자가 먼저 확인해야 할 SEO 기본 요소를 정리합니다.

## 1. 글마다 제목과 설명을 다르게 쓰기

검색엔진은 페이지의 제목과 설명을 보고 내용을 파악합니다. 모든 글의 제목이 비슷하거나 설명이 비어 있으면 검색 결과에서 매력이 떨어집니다.

좋은 제목은 구체적이어야 합니다.

나쁜 예시는 다음과 같습니다.

- AI 이야기
- 블로그 팁
- 자동화 방법

더 나은 예시는 다음과 같습니다.

- ChatGPT로 하루 업무 정리 자동화하는 방법
- GitHub Pages로 무료 블로그를 만드는 이유
- 애드센스 승인용 블로그 체크리스트

제목만 봐도 독자가 무엇을 얻을 수 있는지 보여야 합니다.

## 2. 사이트맵을 제출하기

사이트맵은 검색엔진에게 “이 사이트에는 이런 페이지가 있다”고 알려주는 파일입니다. 정적 블로그에서는 빌드할 때 sitemap.xml이 자동으로 생성되도록 설정하는 것이 좋습니다.

사이트맵이 준비되면 Google Search Console에 등록합니다. 등록 후에는 새 글이 발행될 때 검색엔진이 더 쉽게 발견할 수 있습니다.

## 3. 내부 링크 만들기

블로그 글끼리 연결되어 있으면 방문자가 더 오래 머물고, 검색엔진도 사이트 주제를 이해하기 쉬워집니다.

예를 들어 애드센스 승인 글에서 다음 글로 연결할 수 있습니다.

- 블로그 품질 관리 글
- GitHub Pages 블로그 만들기 글
- SEO 기본 설정 글

내부 링크는 억지로 많이 넣을 필요가 없습니다. 독자가 다음에 읽으면 도움이 되는 글만 자연스럽게 연결하면 됩니다.

## 4. 카테고리를 너무 많이 만들지 않기

초기 블로그에서는 카테고리를 많이 만드는 것보다 몇 개의 핵심 카테고리에 글을 쌓는 것이 좋습니다. 카테고리만 많고 글이 적으면 사이트가 비어 보일 수 있습니다.

처음에는 3~5개 정도가 적당합니다. 이후 글이 충분히 쌓이면 세부 카테고리를 추가해도 늦지 않습니다.

## 5. 모바일에서 읽기 쉬운지 확인하기

검색 유입의 상당수는 모바일에서 발생합니다. 글자 크기, 줄 간격, 버튼 크기, 이미지 폭이 모바일에서 자연스러운지 확인해야 합니다.

SEO는 검색엔진만을 위한 기술이 아닙니다. 결국 읽기 좋은 사이트가 검색에서도 유리합니다. 제목, 설명, 사이트맵, 내부 링크, 모바일 가독성을 먼저 챙기면 GitHub Pages 블로그도 충분히 검색 유입을 만들 수 있습니다.
